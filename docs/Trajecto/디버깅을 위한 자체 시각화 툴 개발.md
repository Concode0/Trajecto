# 디버깅을 위한 자체 시각화 툴 개발

## 1. 문제 인식 : 디버깅의 어려움

Trajecto 프로젝트 개발은 수많은 다변량, 3차원 데이터를 동시에 분석해야 하는 난제에 부딪혔습니다. EKF의 16차원 상태 벡터가 발산할 때, 혹은 ZUPT가 오동작할 때, 매번 시각화를 해서 확인하는 디버깅 방식으로는 그 원인을 찾고 분석하는 것이 매우 어려웠습니다. 이를 해결하기 위해서 Trajecto 프로젝트에 특정화되는 시각화 툴을 설계하기로 했습니다.

## 2. 핵심 설계: "모든 것을 하나의 시간 축에서 본다"

툴의 핵심 목표는 흩어진 모든 데이터(IMU, GT, EKF, TCN)를 하나의 시간 축에서 '동기화'하여 교차 검증하는 것이었습니다. `h5` 형식으로 저장된 각 데이터셋을 로드하여, 다음과 같은 '뷰(View)'를 설계했습니다.

- **[전처리 뷰]** '가속도 기반 DTW'가 올바른 'Warping Path'를 찾았는지, 'Savitzky-Golay 필터'가 노이즈를 효과적으로 제거했는지 시각적으로 검증했습니다.
- **[3D 궤적 뷰]** Ground Truth(iPad), EKF-Only, TCN-Only, 최종 EKF-TCN 궤적을 3D 공간에 오버레이하여 비교했습니다.
- **[알고리즘 검증 뷰]** 3D 펜 자세 애니메이션, IMU 원본 데이터, FSR(ZUPT) 신호, APS(평면 전환) 알고리즘의 내부 상태 변수를 하나의 시간 슬라이더로 동기화했습니다.

## 3. "이 툴이 없었다면 불가능했던 발견들" (핵심 기여)

이 시각화 툴은 단순한 편의 기능이 아니라, 연구의 방향을 결정한 '눈(Eyes)'이었습니다.

1. **"물리적 역설"의 발견:** TCN-Only 궤적이 겉보기엔 그럴싸했지만, 이 툴의 [3D 궤적 뷰]로 프레임 단위로 분석한 결과, 물리 법칙을 무시하는 '순간이동'과 '비현실적 가속'을 '눈으로' 발견할 수 있었습니다. 이는 EKF-TCN 하이브리드 아키텍처를 설계하게 된 결정적 계기였습니다.
2. **ZUPT 및 EKF 튜닝:** [알고리즘 검증 뷰]를 통해, FSR 압력 신호가 0이 되는 '정확한 타이밍'에 EKF의 속도 벡터가 0으로 보정되는지, 그리고 '적응형 R 행렬'이 운동 가속도 발생 시점에 맞춰 R값을 증폭시키는지 실시간으로 확인할 수 있었습니다.
3. **APS 알고리즘 검증:** 3D 펜 자세 애니메이션을 보며 '사분면(Octant)'을 통과하는 횟수를 카운트하는 APS 알고리즘의 임계값을 튜닝할 수 있었습니다.

## 4. 결론

이 Dash 기반의 분석 툴은 Trajecto 프로젝트의 단순한 '결과물'이 아니라, 복잡한 물리/AI 시스템의 내부를 투명하게 들여다보고, 가설을 수립하며, 문제를 해결하는 '연구의 핵심 과정(Methodology)' 그 자체였습니다.
