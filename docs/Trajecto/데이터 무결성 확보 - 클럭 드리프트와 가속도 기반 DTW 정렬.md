# 데이터 무결성 확보 - 클럭 드리프트와 가속도 기반 DTW 정렬

# 문제 인식: 잘못된 짝의 위험성

초기 데이터 획득 시스템은 iPad(Ground Truth)와 Trajecto 보드(Sensor)의 초기 타임스탬프를 기준으로 시간 축을 정렬했습니다. 하지만 이 방식은 두 시스템의 클럭 차이때문에 미세한 오차가 지속적으로 발생하는 문제를 가지고 있습니다.

iPad와 보드의 클럭은 미세하게 속도 차이가 나며, 이 오차는 시간이 지남에 따라서 지속적으로 누적됩니다. 장기간 수집된 데이터의 끝부분은 수백 ms까지 어긋날 수 있었습니다. 이는 실제 모델이 서로 잘못된 짝을 정답으로 판단하는 문제가 발생할 수 있었고, 데이터의 무결성이 해쳐진다고 생각했습니다.

# Hardware Trigger

이 문제를 해결하기 위해서 초기에 고안했던 방안은 하드웨어 트리거를 이용해서 두 시스템의 시간을 물리적으로 묶는 것이었습니다. 그러나 현실적으로 이미 하드웨어와 iPad 어플리케이션 개발이 상당히 완료되어있는 상태에서 이를 수정하는 것은 부담이 있다고 생각했습니다. 그리고 하드웨어 트리거는 초기에는 정확히 정렬하지만 장기간의 클럭 드리프트의 보정에는 소프트웨어적인 보정이 더 유리하다고 판단했습니다. 이와 같은 이유로 소프트웨어적 알고리즘을 설계하기로 했습니다.

# Software Trigger

타임스탬프를 제외하고 다른 데이터를 이용해서 정렬하는 것을 고려했는데, 두 시스템이 공통으로 측정하는 데이터인 가속도를 시간축 정렬에 사용할 수 있도록 설계했습니다.

iPad 신호 - iPad는 2차원 '위치'를 정밀하게 획득합니다. 이를 2번 미분하여 '화면 좌표계'의 2축 가속도를 계산했습니다.

IMU 신호 - 보드의 IMU 데이터에 강체 변환하여, 동일한 '화면 좌표계'로 변환했습니다.

이를 통해서 동일한 '가속도 시계열'을 이용해서 DTW를 수행해 시간축을 정렬할 수 있게 했습니다. 다만, 해당 과정에서 노이즈 문제를 해결하기 위해서 Savitzky-Golay 필터를 적용했습니다. 이를 통해서 클럭드리프트 문제를 강건하게 해결하고 데이터 무결성을 확보했습니다.
