# EKF 모델링 심층 분석: ‘관측 가능성’ 한계와 ZUPT 기반의 강건한 설계

EKF(확장 칼만 필터)는 Trajecto 프로젝트의 물리적 뼈대를 구성하는 핵심 엔진입니다. 저는 EKF를 단순한 라이브러리로 사용하는 '블랙박스' 접근 대신, 물리 법칙으로부터 EKF의 모든 구성 요소를 직접 설계하고 튜닝하는 과정을 거쳤습니다.

### 1. EKF의 근본적 한계: '관측 가능성(Observability)'의 문제

EKF 모델링 초기, 저는 순수 관성 항법(Inertial Navigation)의 근본적인 '이론적 한계'에 부딪혔습니다. 바로 '관측 가능성(Observability)'의 문제입니다. 단일 IMU만으로는 시스템의 모든 상태 변수를 완벽하게 추정할 수 없습니다.

1. **중력과 운동 가속도의 모호성:** 가속도계는 '진짜 운동 가속도'와 '중력'을 구분하지 못합니다. 펜이 등속 직선 운동을 할 때(운동 가속도 = 0), 필터는 가속도계가 측정한 벡터가 (A) 순수 중력인지, (B) 중력 + 보상되지 않은 가속도계 바이어스인지, (C) 자세 오차로 인해 중력 벡터가 잘못 투영된 것인지 명확히 구분할 수 없습니다.
2. **나침반 없는 항해 (Yaw Drift):** 가속도계는 중력(수직 방향)을 기준으로 Roll과 Pitch는 보정할 수 있지만, 수직축 기준의 회전, 즉 '방향(Heading/Yaw)'에 대한 정보는 전혀 제공하지 못합니다. 이는 자이로스코프의 Yaw축 바이어스가 보정되지 않고 계속 누적되어, 궤적이 발산하는 원인이 됩니다.

이 때문에 이론적으로 외부의 기준점 없이는 정확한 궤적을 추적하기는 어렵습니다.

### 2. 핵심 해결책: ZUPT는 '절대 관측치'다

이 '관측 가능성' 문제를 해결한 것은 압력 기반 ZUPT입니다. ZUPT는 현재 속도(`v`)가 [0, 0, 0]이라는 절대적인 기준값을 제공하며, 이는 시스템 외부의 '절대적인 관측치(Absolute Measurement)'로서 활용됩니다.

- (1) ZUPT는 누적된 속도(v) 오차를 직접 제거하여 위치 오차의 발산을 막습니다.
- (2) 정지 상태에서 관측되는 가속값은 '중력 + 바이어스' 뿐이므로, 모호성 없이 가속도계 바이어스를 명확하게 계산할 수 있게 합니다.
- (3) 정확해진 바이어스 값을 바탕으로 드리프트로 흐트러진 자세(Roll/Pitch)를 바로잡**게** 합니다.

### 3. ZUPT 기반의 16차원 상태 벡터 설계

이러한 ZUPT의 존재를 '전제'로, 저는 시스템이 IMU의 정확한 위치 정보를 추적하도록 다음과 같이 16차원 벡터를 설정했습니다.

- `p` (3): 월드 좌표계 기준의 **위치**
- `v` (3): 월드 좌표계 기준의 **속도**
- `q` (4): **쿼터니언** 기반의 자세 (Body to World)
- `b_a` (3): 가속도계 **바이어스**
- `b_g` (3): 자이로스코프 **바이어스**

'짐벌 락' 현상과 완결성을 고려하여 오일러 각 대신 **쿼터니언**을 사용했으며, 이는 강체 변환에도 유리했습니다.

### 4. 바이어스 모델링: ZUPT와 '랜덤 워크'의 시너지

EKF의 성능은 센서 '바이어스'를 얼마나 잘 모델링하느냐에 달려있습니다.

- **(고민 1) 상수 모델:** 'Bias Instability'를 반영하지 못해 기각했습니다.
- **(고민 2) 1차 가우스-마르코프:** 물리적으로 현실적이나, '상관 시간(Tc)' 튜닝이 매우 까다로워 기각했습니다.
- **(결정 3) 랜덤 워크:** 튜닝이 직관적이고(Q_b만 조절), 드리프트를 잘 근사합니다.

이론적으로 Random Walk 모델은 분산이 무한히 발산한다는 단점이 있습니다. 하지만 제 시스템에서는 'ZUPT'가 주기적으로 이 분산을 강제로 리셋시켜줍니다. 따라서 ZUPT와의 시스템적 시너지를 고려할 때, 튜닝이 용이한 Random Walk가 가장 실용적이고 강력한 선택이었습니다.

### **5. 강건한 튜닝: 데이터 기반 $Q$와 적응형 $R$**

• $Q$ **행렬 (프로세스 노이즈):** "내 물리 모델을 얼마나 믿을 것인가?" 이 '감'을 배제하기 위해, 훈련 데이터셋(iPad GT)을 대상으로 ATE/RPE 오차를 최소화하는 값을 찾는 '데이터 기반 최적화'(Grid Search) 과정을 거쳤습니다.
• $R$ **행렬 (측정 노이즈):** "센서 측정값을 얼마나 믿을 것인가?" EKF는 가속도계의 '중력' 관측을 이용해 자세를 보정하는데, 펜을 '휘두를 때'의 '운동 가속도'는 중력 관측을 오염시킵니다.
• **(해결책) 적응형 R 제어:** 이 문제를 해결하기 위해, 가속도계 값의 크기가 중력과 큰 차이가 나는지(운동 중) 실시간으로 감지했습니다. 운동 중임이 확인되면, $R$ 행렬의 값을 100배 증폭시켜 '오염된 측정값'을 무시하도록 설계했습니다. 반대로 ZUPT가 감지되거나 움직임이 거의 없으면 $R$ 값을 낮춰 자세 보정을 적극 수행하도록 했습니다.
