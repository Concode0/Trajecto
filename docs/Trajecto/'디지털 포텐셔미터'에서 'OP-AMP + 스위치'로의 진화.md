# '디지털 포텐셔미터'에서 'OP-AMP + 스위치'로의 진화

## 1. v1의 한계 인식: "디지털 포텐셔미터"의 노이즈와 임피던스 문제

초기 보드 설계에서 저는 FSR의 민감도와 인식 범위를 펌웨어로 조절하기 위해서 ‘디지털 포텐셔미터’를 사용했습니다. 이론적으로는 유연했지만, 실제 테스트 과정에서 두 가지 치명적인 문제에 부딪혔습니다.

1. 신호 노이즈 : 디지털 포텐셔미터는 수많은 스위치로 구성되어, 민감한 아날로그 신호에 디지털 노이즈를 유입시켰습니다.
2. 임피던스 부정합 : FSR은 그 자체로 저항값이 매우 높은 센서인데, 이 약한 신호를 MCU의 ADC 핀에 직접 연결하자 입력 임피던스가 회로에 영향을 주어 신호가 왜곡되는 현상을 발견했습니다. ZUPT 트리거를 위한 미세한 압력 변화를 ‘왜곡 없이’ 읽는 것이 불가능했습니다. 

## 2. v2의 설계 목표: "신호 무결성"과 "데이터 기반 유연성"

v1의 실패를 바탕으로, v2(4-Layer PCB)의 FSR 회로는 두 가지 명확한 목표를 가지고 재설계되었습니다.

1. **신호 무결성(Signal Integrity) 확보:** FSR의 약한 아날로그 신호를 '왜곡 없이' ADC로 전달할 것.
2. **데이터 기반 유연성(Data-Driven Flexibility):** v1 실험에서 찾은 '최적의 저항값'들을 안정적으로 선택할 수 있을 것.

## **3. v2의 핵심 솔루션: "OP-AMP 버퍼"와 "아날로그 스위치"**

이 두 가지 목표를 달성하기 위해, 저는 회로를 '디지털 포텐셔미터' 대신 "OP-AMP"와 "아날로그 스위치"의 조합으로 완전히 재설계했습니다.

- **1. OP-AMP (신호 무결성 확보):**
    - FSR 전압 분배기 회로와 ADC 핀 사이에 'OP-AMP를 이용한 전압 팔로워(Voltage Follower, 버퍼)'를 삽입했습니다.
- **2. 아날로그 스위치 (데이터 기반 유연성):**
    - v1 프로토타입을 테스트하며, 저는 "가벼운 터치(펜 그립)"과 "필기 상황 터치(강한 그립)"을 감지하는 데 최적화된 저항값이 다르다는 것을 실험적으로 알아냈습니다.
    - v2에서는 이 v1의 실험 데이터(1MΩ, 10kΩ)를 기반으로, 펌웨어가 이 두 개의 고정 저항을 선택할 수 있도록 '아날로그 스위치'를 배치했습니다.
    - 이는 노이즈가 많은 '디지털 포텐셔미터' 대신, '안정적인 고정 저항'을 사용함으로써 신호의 품질을 유지하면서도 시스템의 민감도를 2-Mode로 조절할 수 있는 최적의 설계였습니다.

## 4. 결론

v1의 '디지털 포텐셔미터'는 아이디어 차원의 접근이었지만, 'OP-AMP 버퍼'와 '아날로그 스위치'를 도입한 v2 회로는 실제 센서의 물리적 특성(고임피던스)과 실험 데이터(최적 저항값)에 기반한 '공학적 솔루션'입니다. 이 설계를 통해 ZUPT 알고리즘과 TCN 모델이 신뢰할 수 있는 고품질 압력 데이터를 확보하는 기반을 마련했습니다.
